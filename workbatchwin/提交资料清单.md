# 题目 3 提交资料清单（workbatchwin）

提交到平台时，请将 `workbatchwin/` 目录内容作为最终 `work/` 目录提交。

## 1. 必须提交

```text
work/
  INSTRUCTION.md
  技术报告.md
  提交资料清单.md
  WINDOWS_README.md
  self_check.py
```

说明：

```text
INSTRUCTION.md        裁判执行说明书，平台会优先读取
技术报告.md           技术方案、部署手册、基准验证说明
提交资料清单.md       当前文件，便于提交前核对
WINDOWS_README.md     Windows 快速使用说明
self_check.py         本地烟测脚本，可选执行
```

## 2. OpenCode 组件

必须提交：

```text
work/.opencode/
  commands/spec-audit.md
  commands/prepare-rfcs.md
  agents/spec-compliance-orchestrator.md
  agents/code-investigator.md
  agents/evidence-reviewer.md
  tools/audit_start.ts
  tools/audit_next.ts
  tools/audit_finish.ts
  tools/code_search.ts
  tools/prepare_rfcs.ts
  tools/submit_batch_results.ts
  tools/submit_conclusion.ts
  tools/submit_investigation.ts
  tools/submit_review.ts
  tools/audit_dispatch_result.ts
  skills/spec-code-consistency/SKILL.md
```

这些文件是 OpenCode 交互式主流程。评测时从 `work/` 启动 OpenCode，并通过 `/spec-audit` 的第一个参数传入被审计代码仓路径。

## 3. 平台 Skill 路径

平台要求如果包含 Skill，路径必须为：

```text
work/skills/{your-skill-name}/SKILL.md
```

本作品对应：

```text
work/skills/spec-code-consistency/SKILL.md
work/skills/spec-code-consistency/resources/
  finding_schema.md
  investigation_schema.md
  review_checklist.md
  taxonomy.md
```

## 4. SpecDiff Runtime

必须提交：

```text
work/specdiff/
```

核心文件包括：

```text
audit_runtime.py
tool_api.py
rfc_prepare.py
rfc_packs.py
repository_index.py
codefacts.py
spec_loader.py
coverage_gate.py
report.py
models.py
```

该 runtime 由 `.opencode/tools/*.ts` 通过：

```text
python -m specdiff.tool_api
```

调用。

## 5. 离线/精简依赖

建议提交：

```text
work/specdiff-vendor-slim/
```

用途：

```text
提供精简 Tree-sitter / networkx 等依赖
降低评测环境缺包导致失败的风险
```

## 6. Windows 版部署副本

建议保留：

```text
work/.opencode/specdiff-runtime/
```

这是已打包好的 OpenCode runtime 副本，OpenCode tools 会从当前项目根目录加载：

```text
work/.opencode/specdiff-runtime/
```

## 7. 测试与回归资料

建议提交：

```text
work/tests/
work/testdata/
```

用途：

```text
本地单元测试
自检用小型样例
输出格式与流程回归材料
```

测试命令：

```powershell
cd work
$env:PYTHONPATH = "."
python -m unittest discover -s tests
```

## 8. 平台提供，不需要提交

```text
code/f-stack/
Difference/benchmark.md
Difference/issues/

```

这些由评测平台提供，提交包中不应重复包含完整 F-Stack 代码仓。
